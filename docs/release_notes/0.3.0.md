### Structural Changes

New models:
- `Section` - all modules are now grouped in sections
- `SectionConfig` / `ModuleConfig` - breakpoint-specific settings for sections/modules
- `SectionSymlink` - links an existing section into another container; accessible via `container.sectionsymlinks`; `container.get_sections_front()` returns a sorted Python list combining `Section` and `SectionSymlink`

Module API changes:
- `extra_context` → `context` (now required in `render_front()` / `render_admin()`)
- `content_type` now uses full model path (e.g. `xprez.TextModule`)
- `Module.saved` / `Section.saved` - unsaved items not rendered on frontend
- Deletion happens after form submit, not immediately
- `ModuleSymlink` is now auto-registered in `XPREZ_MODULES_AUTOREGISTER_BUILTINS`; excluded from the add-menu by default via `XPREZ_MODULES_ADD_MENU_EXCLUDE`

Admin URL name changes:
- `clipboard_copy` → `clipboard_clip` (path: `xprez-clipboard-copy/` → `xprez-clipboard-clip/`)
- `copy_module` / new `duplicate_section` → unified `duplicate` (path: `xprez-copy-module/` / `xprez-duplicate-section/` → `xprez-duplicate/<key>/<pk>/`)
- Clipboard paste / list URL parameters changed from position-based (`target_position`, `target_pk`) to container/section PKs (`target_container_pk`, optional `target_section_pk`)

Admin view response format:
- All add and duplicate views now return `JsonResponse([{"html": "..."}], safe=False)` instead of `HttpResponse` with raw HTML
- Clipboard paste response changed from `{"modules": [...], "updated_module_positions": {...}}` to `[{"html": "..."}, ...]`

Frontend JavaScript:
- Admin JS fully rewritten from jQuery-based (`contents.js`, `ajax_upload_formset.js`) to plain ES modules (`xprez.js`); any project that extended or monkey-patched the old classes must rewrite those customisations

Removed:
- `Module.show_front()` - override `render_front()` instead
- `Module.page` - use `Module.section.container`
- `Module.create_for_page()` - use `{Model}.objects.create()`
- `Content.copy(for_page, position, save)` - use `Module.duplicate_to(target_section)`
- `ContentsContainer.copy_contents(for_container)` - use `Container.duplicate_to(target_container)`
- `ContentItem.copy(for_content, save)` - use `MultiModuleItem.duplicate_to(target_module)`
- `FormsetContent.copy_items(inst)` - use `MultiModule.duplicate_items(new_module)`
- `XprezAdminMixin.xprez_add_content_before_*()` - use `add_view()`
- `XprezAdminMixin.xprez_ui_css_class`
- `XprezAdminMixin.copy_*` methods - use clipboard instead
- `XprezAdminMixin.allowed_contents` / `excluded_contents` class attrs - use `XPREZ_MODULES_ALLOWED` / `XPREZ_MODULES_ALLOWED_EXCLUDE` settings
- `xprez_get_allowed_contents(container)` / `xprez_get_allowed_content_types(container)` - use `xprez_allowed_modules()` / `xprez_allowed_module_classes()`
- `xprez_clipboard_copy_url_name()` → `xprez_clipboard_clip_url_name()`
- `XprezModelFormMixin.save_xprez_contents(request)` → `xprez_save(request)`
- `XprezModelFormMixin.xprez_contents` attribute → `xprez_sections`
- `POSITION_CONTENT_BEFORE` / `POSITION_CONTAINER_END` constants - module/section position is now fully managed by the frontend and written to hidden form inputs on save; the backend no longer accepts or acts on a position in add/paste URLs
- `CLIPBOARD_CONTENT_KEY` value was `"content"` → now `"module"` (existing session data will be ignored after upgrade)

Removed sub-apps (remove from `INSTALLED_APPS`):
- `xprez.code_snippet` - removed entirely; `CodeSnippet` content type is gone
- `xprez.medium_editor` - removed entirely; `MediumEditor` content type is gone
- `xprez.ck_editor` - no longer a standalone Django app; remove from `INSTALLED_APPS` (the module still exists for widgets/fields)

Moved to **xprez.admin**:
- `xprez.admin_forms` → `xprez.admin.forms`
- `xprez.form_fields` / `xprez.models.fields` → `xprez.admin.fields`
- CK editor widget template: `xprez/widgets/ck_editor.html` → `xprez/admin/widgets/ck_editor.html`

Structural reorganization:
- Module components moved to `xprez/modules/` (model + config + form per module)
- `models/`: `configs`, `containers`, `modules` (Module), `multi_module` (MultiModule, MultiModuleItem, UploadMultiModule), `sections`; `mixins/` (css, font_size, responsive_image)
- `admin/forms.py` reduced to base forms only
- CSS utilities: `xprez.models.mixins.css`
- Responsive image mixins: `xprez.models.mixins.responsive_image`
- `FontSizeModuleMixin`: `xprez.models.mixins.font_size`
- `MultiModule`, `MultiModuleItem`, `UploadMultiModule`: `xprez.models.multi_module`
- Media collection: admin and front media both use a collector that takes initial JS/CSS from settings, then adds media from each module’s `AdminMedia` / `FrontMedia` class. Front media now has `XPREZ_FRONT_MEDIA_JS` and `XPREZ_FRONT_MEDIA_CSS` (defaults: empty tuple and empty dict); same pattern as `XPREZ_ADMIN_MEDIA_JS` / `XPREZ_ADMIN_MEDIA_CSS`.

### MultiModule: individual forms instead of formsets

`FormsetContent` (now `MultiModule`) no longer uses Django's `inlineformset_factory`. Each item now gets its own standalone `ModelForm` keyed by a stable per-item prefix (`item-module-{module_pk}-{item_pk}`).

**Migration guide for custom `FormsetContent` subclasses:**

- Rename `form_class` class attribute to `admin_form_class` (string import path to a `ModuleForm` subclass).
- Remove `formset_factory` class attribute; override `admin_item_form_class` (string import path to a `MultiModuleItemForm` subclass) instead.
- Override `admin_item_template_name` (string path) for a custom item row template; override `admin_template_name` for the module’s admin wrapper (e.g. `UploadMultiModule` uses a different wrapper).
- Remove `get_formset_queryset()`; override `get_items_queryset()` if custom ordering or filtering is needed.
- Replace any `self.formset` access with `self.admin_form.xprez_items` (a list of item instances, each with `.admin_form` attached).
- Delete handling moves from formset's `can_delete=True` to `DeletableFormMixin` on the item form; items with `delete=True` in cleaned data are deleted in `save_admin_form`.
- Items created via "Add item" are inserted with `saved=False`; they are excluded from frontend rendering until the page form is submitted and saved.
- `copy_items(inst)` → `duplicate_items(new_module)`.

**AJAX add-item / upload response format changed:**

Previously the server returned a two-piece JSON dict (`{"form": ..., "template": ...}`). Now it returns a single raw HTML fragment (the fully rendered item template including its form fields). JS no longer needs to patch management form counters.

**JS dependency change:**

The old `ajax_upload_formset.js` (jQuery + Dropzone.js) is replaced by `multi_module.js` — a plain ES module. Custom JS that extended or called into the old upload/formset logic must be rewritten against `XprezMultiModule` / `XprezUploadMultiModule`.
