0.3.0
=====

- introduce `Section` model. All contents are now in sections.
- introduce `SectionConfig` models:
    - holds css-breakpoint specific settings for each section
- introduce `ContentConfig` models:
    - holds css-breakpoint specific settings for each content
    - specified by `Content.config_model` for each `Content`. (e.g. config_model = `xprez.TextContentConfig`)
- renamed `extra_context` parameter to `context` and is now required in:
    - `Content.render_front()`
    - `Content.render_admin()`
- removed `Content.show_front()`. Override `Content.render_front()` instead.
- removed `Content.page`. Use `Content.section.container` instead.
- removed `Content.create_for_page()`. Use standard `{Model}.objects.create()`, position is set by frontend.
- `Content.content_type` is now full model class following django convention
    - e.g. `xprez.TextContent`, not `textcontent`
    - this is now the (only) way to specify contents - in `settings` or `allowed_contents`. 
- removed `XprezAdminMixin.xprez_add_content_before_*()`.  Now handled by `XprezAdminMixin.add_view()`.
- attributes `Content.saved` and `Section.saved`
    - indicates if content/section has been saved in admin
    - unsaved contents/sections are not rendered in frontend
